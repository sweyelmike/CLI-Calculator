### üéØ Core Objectives

The primary goal of this project was to demonstrate proficiency in writing **clean, modular, and maintainable Python code**. Specifically, the implementation leverages:

  * **Functions:** To encapsulate and manage each arithmetic operation.
  * **Loops:** To ensure the calculator runs continuously until the user explicitly exits.
  * **Conditionals (Control Flow):** For decision-making based on user input (operation selection and continuation choice).

-----

### ‚ú® Key Features

  * **Four Fundamental Operations:** Supports `+` (Addition), `-` (Subtraction), `*` (Multiplication), and `/` (Division).
  * **Modular Design:** Each mathematical operation is implemented as a dedicated, reusable function.
  * **Continuous Operation:** The application utilizes a loop to allow multiple calculations without restarting.
  * **User-Friendly CLI:** Simple, text-based interaction for easy use by beginners.

### ‚öôÔ∏è Application Workflow

The program follows a straightforward execution cycle:

1.  The user is prompted to input **two numbers** and select the desired **arithmetic operator**.
2.  The program utilizes **conditional logic** to call the corresponding function (e.g., `add()`, `subtract()`, etc.) based on the operator.
3.  The calculated **result** is immediately displayed to the user.
4.  Finally, the user is asked whether they wish to **continue** with another calculation or **exit** the application.
